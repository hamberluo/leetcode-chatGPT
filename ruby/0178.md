题目编号：178

题目：分数排名

难度：中等

考察范围：SQL

题干：编写一个 SQL 查询来实现分数排名。如果两个分数相同，则两个分数排名相同。请注意，平分后下一个名次应该跳过相同数量的名次。

表：

```ruby
+----+-------+
| Id | Score |
+----+-------+
| 1  | 3.50  |
| 2  | 3.65  |
| 3  | 4.00  |
| 4  | 3.85  |
| 5  | 4.00  |
| 6  | 3.65  |
+----+-------+
```

例如，根据上述给定的 Scores 表，你的查询应该返回（按分数从高到低排列）：

```ruby
+-------+------+
| Score | Rank |
+-------+------+
| 4.00  | 1    |
| 4.00  | 1    |
| 3.85  | 2    |
| 3.65  | 3    |
| 3.65  | 3    |
| 3.50  | 4    |
+-------+------+
```

解题思路：使用 MySQL 的内置函数 rank() 和 dense_rank() 可以轻松实现排名功能。其中 rank() 函数会跳过相同的排名，而 dense_rank() 函数则不会跳过。

解决方案：

```ruby
SELECT Score, DENSE_RANK() OVER (ORDER BY Score DESC) AS Rank
FROM Scores
ORDER BY Score DESC;
```

算法复杂度：时间复杂度 O(nlogn)，空间复杂度 O(n)。
