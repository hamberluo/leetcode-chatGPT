题目编号：789

题目：逃脱阻碍者

难度：中等

考察范围：图论、BFS

题干：你在进行一个简化版的吃豆人游戏。你从 (0, 0) 点开始出发，你的目的地是 (target[0], target[1]) 。地图上有一些阻碍者，第 i 个阻碍者从 (ghosts[i][0], ghosts[i][1]) 出发。每个阻碍者每秒可以向上下左右四个方向移动一个单位长度。如果你和阻碍者同时到达了某一个位置，那么游戏结束。你可以在任意时刻停止移动，而阻碍者则不能停止移动。现在，你的任务是判断在你是否能够在不和阻碍者相遇的情况下到达目的地。

解题思路：本题可以使用 BFS 算法来解决。我们可以将起点、终点和阻碍者的位置都看作是图中的节点，然后使用 BFS 算法来搜索从起点到终点的最短路径。在搜索的过程中，我们需要判断当前节点是否与阻碍者重合，如果重合则直接返回 false，否则继续搜索。最后如果能够搜索到终点，则返回 true，否则返回 false。

解决方案：

```ruby
def escape_ghosts(ghosts, target)
  # 计算起点到终点的曼哈顿距离
  distance = (target[0] - 0).abs + (target[1] - 0).abs

  # 遍历每个阻碍者，计算它到终点的曼哈顿距离
  ghosts.each do |ghost|
    ghost_distance = (ghost[0] - target[0]).abs + (ghost[1] - target[1]).abs

    # 如果有任意一个阻碍者到终点的距离小于等于起点到终点的距离，则无法逃脱
    return false if ghost_distance <= distance
  end

  # 如果所有阻碍者到终点的距离都大于起点到终点的距离，则可以逃脱
  true
end
```

算法复杂度：时间复杂度为 O(n)，其中 n 为阻碍者的数量。空间复杂度为 O(1)。
