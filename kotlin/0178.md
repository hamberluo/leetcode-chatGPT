题目编号：178

题目：分数排名

难度：中等

考察范围：排序、SQL

题干：编写一个 SQL 查询来实现分数排名。如果两个分数相同，则两个分数排名相同，排名应该从高到低排列。

表格 scores：
```kotlin
+----+-------+
| Id | Score |
+----+-------+
| 1  | 3.50  |
| 2  | 3.65  |
| 3  | 4.00  |
| 4  | 3.85  |
| 5  | 4.00  |
| 6  | 3.65  |
+----+-------+
```
例如，根据上述给定的表格，你的查询应该返回（按分数从高到低排列）：
```kotlin
+-------+------+
| Score | Rank |
+-------+------+
| 4.00  | 1    |
| 4.00  | 1    |
| 3.85  | 2    |
| 3.65  | 3    |
| 3.65  | 3    |
| 3.50  | 4    |
+-------+------+
```

解题思路：使用 SQL 语句，先按照分数从高到低排序，然后使用变量记录排名，如果分数相同则排名不变，否则排名加一。

解决方案：
```kotlin
SELECT Score, 
       @rank := @rank + (@prev <> (@prev := Score)) Rank
FROM scores, (SELECT @rank := 0, @prev := -1) init
ORDER BY Score DESC;
```

算法复杂度：时间复杂度 O(nlogn)，空间复杂度 O(1)。
