题目编号：161

题目：一次编辑

难度：中等

考察范围：字符串、双指针

题干：给定两个字符串 s 和 t，判断它们是否只需要进行一次（或者零次）编辑就可以变成相同的字符串。

一次编辑包括以下 3 种操作：

1. 插入一个字符
2. 删除一个字符
3. 替换一个字符

解题思路：双指针

我们可以使用双指针的方法来解决这个问题。首先，我们可以判断两个字符串的长度差是否大于 1，如果大于 1，那么它们一定不能通过一次编辑变成相同的字符串。如果长度差小于等于 1，那么我们可以使用双指针分别指向两个字符串的开头，然后比较两个指针所指向的字符是否相同。如果相同，那么两个指针都向后移动一位；如果不同，那么我们可以尝试进行一次编辑操作，具体来说，我们可以尝试删除 s 中指针 i 所指向的字符，或者在 t 中插入一个字符，使得 s 和 t 中指针 i 所指向的字符相同。如果我们进行了一次编辑操作，那么我们就将指针 j 向后移动一位，否则，我们就将两个指针都向后移动一位。如果最终两个指针都到达了字符串的末尾，那么这两个字符串可以通过一次编辑变成相同的字符串。

解决方案：

```java
class Solution {
    public boolean isOneEditDistance(String s, String t) {
        int m = s.length(), n = t.length();
        if (Math.abs(m - n) > 1) {
            return false;
        }
        int i = 0, j = 0;
        boolean edited = false;
        while (i < m && j < n) {
            if (s.charAt(i) == t.charAt(j)) {
                i++;
                j++;
            } else {
                if (edited) {
                    return false;
                }
                edited = true;
                if (m > n) {
                    i++;
                } else if (m < n) {
                    j++;
                } else {
                    i++;
                    j++;
                }
            }
        }
        return edited || i < m || j < n;
    }
}
```

算法复杂度：时间复杂度为 O(n)，其中 n 是两个字符串的长度之和。空间复杂度为 O(1)。
